Keis — Replit AI Landing Generator (инструкции)

Цель: минимальные действия с вашей стороны: форк/создать Repl → вставить секреты → Run → редактировать бриф → Сгенерировать → Скачать ZIP.

Файлы в проекте:
- index.html — клиентский интерфейс (редактируете бриф, выбираете шаблон, смотрите превью, экспорт).
- server.js — Express-сервер: вызывает OpenAI, хранит результат в памяти, формирует ZIP.
- package.json — зависимости.
- .replit — команда запуска.
- create_zip.sh — (опционально) для локального архива.

Что нужно сделать (5–10 минут):
1) Зарегистрируйтесь / войдите в Replit: https://replit.com
2) Создайте новый Repl:
   - Нажмите "+ Create" -> "Import from GitHub" (если у вас есть репо) или "Create Repl" -> выберите "Node.js".
   - Если создаёте пустой Node Repl: удалите default файлы, добавьте файлы из этого шаблона (index.html, server.js, package.json, .replit).
3) В Replit GUI настройте Secrets (Environment variables):
   - OPENAI_KEY = ваш ключ (sk-...)
   - ALLOWED_TOKEN = любой короткий токен (например: keis-widget-1). Если не хотите токен — оставьте пустым (но это менее безопасно).
   (В Replit: Tools -> Secrets)
4) Нажмите "Run" — сервер запустится. В правом окне Replit откроется веб-приложение.
5) Использование:
   - Введите сессию (например session-1), опишите бриф, нажмите "Сгенерировать".
   - Сервер вызовет OpenAI, вернёт JSON, клиент покажет превью.
   - Нажмите "Скачать ZIP" — получите архив landing.html + landing.json.
6) Загрузка в Тильду:
   - Откройте архив, откройте landing.html, скопируйте HTML в HTML-блок Тильды или загрузите файл как страницу (в зависимости от workflow).
7) Приглашение/коллаборация:
   - В Replit можно пригласить коллегу в проект (Share -> Invite) — тогда второй человек увидит код и интерфейс. OpenAI будет работать через сервер (ваш ключ).

Безопасность:
- OPENAI_KEY храните только в Replit Secrets.
- ALLOWED_TOKEN используется для проверки при экспорте и вызовах (опционально).
- Для production: замените inMemoryStore на базу (Redis/Firestore) и добавьте rate limits.
